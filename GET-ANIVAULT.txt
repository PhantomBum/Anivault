GET ANIVAULT UNVAULTED (Windows)
================================

WHERE TO DOWNLOAD
-----------------
Official repo (bookmark this; forks may not have builds):

  https://github.com/PhantomBum/Anivault/releases

Open the latest release. Under Assets you want at least:

  AniVaultUnvaultedSetup.exe     — run this to install

You’ll also see Squirrel updater files on the same release (e.g. *-full.nupkg, RELEASES, latest.yml).
The in-app updater expects those to live alongside the setup on GitHub.

If Assets only shows “Source code (zip/tar.gz)”, that tag never got a successful Windows build.
Pick another release or check Actions for a red run.


SHIP A NEW VERSION (maintainers)
--------------------------------
From the repo root in PowerShell:

  .\scripts\release-desktop.ps1 1.2.5

Use a new version number each time. That bumps desktop/package.json, commits, pushes main, tags
v1.2.5, and pushes the tag. Wait for “Release desktop (Windows)” to go green, then refresh the
Releases page — the .exe and updater files should be attached.

You can also bump package.json yourself, push main, then:

  git tag v1.2.5
  git push origin v1.2.5


RE-RUN CI WITHOUT A NEW TAG
---------------------------
GitHub → Actions → “Release desktop (Windows)” → Run workflow → branch main.
Builds whatever version is already in desktop/package.json on that branch.


TEST IN-APP UPDATES LOCALLY
---------------------------
1) Install an older release from GitHub.
2) Publish a newer release with CI green and a higher version.
3) Open the old app → Settings → Updates → check for updates.

Optional logging:

  set ANIVAULT_UPDATER_DEBUG=1
  (then launch the installed exe from that same cmd window)

Install path is usually under %LOCALAPPDATA% for Squirrel apps; Start menu shortcut →
open file location is the lazy way to find it.


REMINDER
--------
Pushing to main only updates source. Installers show up after a green release workflow — almost
always tied to a version tag like the steps above.
